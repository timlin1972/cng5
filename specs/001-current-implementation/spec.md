# Feature Specification: 現有系統能力總覽（Current Implementation Baseline）

**Feature Branch**: `001-current-implementation`

**Created**: 2026-07-24

**Status**: Draft

**Input**: User description: "現在的實作"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 跨網域安全遠端操控其他機器 (Priority: P1)

使用者在自己手邊的一台機器上，就能對位於完全不同網路環境（不同公司網路、不同 NAT
之後、彼此無法直接連線）的另一台機器下指令、查看它的即時狀態面板、或雙向同步指定
資料夾內容，而不需要那台遠端機器對外開放任何連接埠，也不需要兩端在同一個區域網路。

**Why this priority**：這是整個系統存在的核心理由——讓使用者可以安全地「跨網域遙控
自己散落各處的機器」，其餘功能（多介面操作、裝置群組、各項小工具）都是為了讓這件事
用起來更順手。

**Independent Test**：在兩台位於不同網路、彼此無法直接連線的機器上，各自佈署好相同
的共用金鑰並設定好相同的橋接代號後，從其中一台對另一台下一個查詢指令，能收到正確
回應；即可視為此能力獨立可用，不需要其他功能配合。

**Acceptance Scenarios**：

1. **Given** 兩台機器分屬不同網域、彼此網路互不可達，但都已設定相同的橋接代號與共用
   金鑰檔案，**When** 使用者在其中一台上輸入指令連線到另一台的識別碼，**Then**
   連線成功後使用者輸入的每一行指令都會被送到遠端機器執行，且提示字元會顯示目前
   正連線到哪一台機器。
2. **Given** 使用者已跨網域連線到遠端機器，**When** 使用者請求複製一個允許分享的
   資料夾內容，**Then** 檔案會以加密、分段方式傳輸過來，且不會刪除本地既有、內容
   相同的檔案。
3. **Given** 一則跨網域訊息在傳輸過程中被延遲或被重複發送，**When** 該訊息送達
   接收端時，其時間戳已超過允許的時間窗、或與先前處理過的訊息重複，**Then** 該
   訊息會被拒絕，不會被誤當成有效指令執行。

### User Story 2 - 單一狀態、多種介面同時操作同一台機器 (Priority: P2)

使用者可以依當下情境，任意選擇文字指令列（終端機 CLI）、終端機內的可視化面板
（TUI）、或瀏覽器網頁介面來操作同一台機器，三者看到與變更的是同一份即時狀態
（指令歷史、外掛面板內容、目前設定），切換介面不會遺失現況或造成不同步。

**Why this priority**：多介面共用同一份狀態是後續所有情境（跨網域操控、裝置群組
查詢、各項工具）能夠一致運作的基礎，也是使用者日常最常互動的部分。

**Independent Test**：在同一台機器上，先透過瀏覽器開啟某個外掛面板，再透過終端機
CLI 或 TUI 對同一個外掛下操作指令，能在瀏覽器畫面上即時看到變化，即可視為此能力
獨立可用。

**Acceptance Scenarios**：

1. **Given** 使用者已透過終端機介面進入某個外掛並顯示其面板，**When** 使用者改
   用瀏覽器開啟同一台機器的網頁介面，**Then** 瀏覽器能看到該面板目前的內容，並
   在內容持續變化時即時更新，不需要手動重新整理。
2. **Given** 使用者在終端機圖形化面板中同時開啟多個外掛面板，**When** 使用者調整
   某個面板的位置、大小、顯示或隱藏狀態，**Then** 畫面立即反映新的配置，且透過
   瀏覽器查詢同一個外掛時看到的內容與終端機一致。
3. **Given** 使用者不在終端機前（例如程式以非前景方式啟動），**When** 使用者仍需
   要操作機器，**Then** 使用者可以完全透過瀏覽器介面完成指令執行與狀態查看，不
   依賴終端機介面存在。

### User Story 3 - 掌握群組內所有機器的上線狀態 (Priority: P3)

使用者可以把多台機器組成一個群組（一台作為伺服器、其餘作為客戶端），之後在群組
中的任何一台機器上，都能查到群組內所有機器目前的基本資訊與是否有定期回報，不需要
逐一登入每台機器確認。

**Why this priority**：這是啟用「跨網域遠端操控」與「批次管理」前，使用者需要先
建立的群組意識與可視性；沒有裝置清單，使用者無法知道有哪些機器可以被操控。

**Independent Test**：把一台機器設為伺服器、另一台設為客戶端並指向該伺服器後，
在客戶端查詢裝置清單，能看到伺服器與自己都列在其中且基本資訊正確，即可視為此能力
獨立可用。

**Acceptance Scenarios**：

1. **Given** 一台機器被設定為伺服器、另一台設定為客戶端並指向該伺服器，**When**
   客戶端啟動並運作一段時間後，**Then** 在伺服器與客戶端上查詢裝置清單，都能看到
   彼此的基本資訊（作業系統、內網位址、版本、目前角色、運作時間）。
2. **Given** 群組內某台客戶端已經很久沒有回報，**When** 使用者查詢裝置清單，
   **Then** 該機器的最近回報時間能讓使用者判斷它是否還在運作，而不是被當成剛剛
   才上線。

### User Story 4 - 個人化維運小工具 (Priority: P4)

使用者可以在同一套介面中，使用一系列日常維運與生活相關的小工具：喚醒區域網路內
指定機器、查詢天氣、記錄與編輯 Markdown 筆記、下載並管理個人音樂庫、產生可供手機
掃描的連線 QR Code、以及監控多個本機 git 專案是否有未提交的變更。

**Why this priority**：這些工具彼此獨立、各自都能單獨帶來價值，但都不是系統存在
的核心理由，屬於在核心能力之上附加的便利功能。

**Independent Test**：任選一項工具（例如天氣查詢）獨立操作並取得預期結果，不需要
其他工具或跨機器功能配合，即可視為此工具獨立可用。

**Acceptance Scenarios**：

1. **Given** 使用者已登記一台機器的名稱與硬體位址，**When** 使用者對該名稱送出
   喚醒指令，**Then** 對應機器會收到喚醒訊號。
2. **Given** 使用者已登記一或多個城市，**When** 使用者查看天氣面板，**Then**
   畫面顯示各城市目前與未來數天的天氣資訊，且資訊不會頻繁重新查詢外部服務。
3. **Given** 使用者在筆記面板中輸入 Markdown 內容，**When** 使用者輸入標題、
   清單、程式碼區塊等語法，**Then** 除了游標所在行以外的內容會以排版後的樣式
   呈現。
4. **Given** 使用者登記了本機一或多個 git 專案路徑，**When** 使用者查看專案監控
   面板，**Then** 有未提交變更的專案會被標示出來。

### User Story 5 - 免手動介入的自我更新 (Priority: P5)

使用者可以下達一個更新指令，系統會自動取得最新版本、重新編譯並在成功後自動切換
成新版本運作，過程中既有的背景服務（如網頁介面、裝置回報）不會中斷；若編譯失敗，
系統會維持在目前可運作的版本上。

**Why this priority**：這是維運便利性功能，讓使用者不需要手動 SSH 進每台機器
更新，但系統即使沒有這個功能，其餘能力仍然完整可用。

**Independent Test**：對單一機器下達更新指令，能觀察到系統在背景完成更新流程並
於成功後切換版本，即可視為此能力獨立可用。

**Acceptance Scenarios**：

1. **Given** 使用者對本機下達更新指令，**When** 更新流程在背景執行，**Then**
   使用者仍可繼續使用目前版本操作機器，直到新版本準備就緒。
2. **Given** 更新流程建置新版本失敗，**When** 建置結束，**Then** 系統維持在
   目前版本繼續運作，不會切換到無法運作的版本。
3. **Given** 使用者透過跨網域或群組連線對遠端機器下達更新指令，**When** 指令送達
   遠端機器，**Then** 更新流程在遠端機器上執行，且本地端的連線工作階段會自動結束。

---

### Edge Cases

- 跨網域訊息的時間戳超過允許的時間窗（過舊或來自時鐘不同步的機器）時，該訊息 MUST
  被拒絕而非被容忍執行。
- 同一則跨網域訊息被重複送達（網路重送、惡意重放）時，第二次以後 MUST 被拒絕。
- 兩端共用金鑰檔案內容不一致時，雙方 MUST 無法互相解密對方訊息，且此情況應可被
  使用者察覺（而非默默失敗、誤判為對方離線）。
- 使用者嘗試複製不在允許清單內的資料夾時，該操作 MUST 被拒絕。
- 使用者針對同一個檔案複製工作再次發起複製、或前一個複製工作尚未完成時，系統如何
  處理同時發起的第二個複製請求需要有明確且一致的行為（例如：拒絕新請求並顯示目前
  進度）。
- 使用者嘗試喚醒一台已經開機的機器時，系統不會回報錯誤，也不會造成任何副作用。
- 群組伺服器機器離線或不可連線時，客戶端如何得知自己暫時無法取得最新裝置清單，
  需要有明確行為而非靜默呈現過期資料且不告知使用者資料已過期。
- 瀏覽器介面與終端機介面幾乎同時對同一份筆記進行編輯並儲存時，最終保留的內容需
  有明確、可預期的規則（例如：最後一次儲存覆蓋先前內容）。
- 自我更新流程在下載/建置期間，若使用者需要用到既有功能（如查看裝置清單、操作
  其他外掛），這些既有功能 MUST 不受影響持續可用。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 讓使用者透過至少三種介面（終端機文字指令列、終端機圖形化
  面板、瀏覽器網頁）操作同一台機器，且三者讀取與變更的是同一份共用狀態。
- **FR-002**: 系統 MUST 提供可插拔的功能模組（外掛）架構，每個模組可各自定義自己
  支援的指令與可視面板內容。
- **FR-003**: 每個功能模組的可視面板 MUST 可被獨立顯示、隱藏、移動、調整大小，且
  多個面板可同時開啟並疊放。
- **FR-004**: 系統 MUST 提供一份可查詢的裝置清單，記錄群組內每台機器的基本資訊
  （作業系統、內網位址、版本、目前角色、運作時間、最近回報時間）。
- **FR-005**: 系統 MUST 支援機器以「獨立」「群組伺服器」「群組客戶端」三種角色之
  一運作；群組客戶端 MUST 定期向所屬伺服器回報自身狀態，並取得群組內完整裝置清單。
- **FR-006**: 系統 MUST 支援把多個群組（網域）透過共同的橋接代號串接起來，使不同
  群組的裝置清單能互相可見。
- **FR-007**: 系統 MUST 讓使用者在同一群組內的機器之間，直接以識別碼連線並執行
  遠端指令，如同操作本機一般。
- **FR-008**: 系統 MUST 讓使用者在已橋接的不同群組（網域）之間，以「網域/識別碼」
  的形式連線並執行遠端指令、查看遠端面板，即使雙方網路互不可達。
- **FR-009**: 所有跨網域傳輸的訊息內容 MUST 經過加密，且訊息 MUST 攜帶時間戳；
  超出允許時間窗或屬於重複訊息的訊息 MUST 被拒絕處理。
- **FR-010**: 系統 MUST 讓使用者在群組內或跨網域的機器之間，把指定且經過允許的
  資料夾內容複製到本機或推送到對方機器，且複製過程 MUST 只新增檔案、不刪除既有
  檔案，並跳過內容已相同的檔案。
- **FR-011**: 系統 MUST 提供一份可查詢的活動紀錄，記錄本機與外部（其他機器、外部
  服務）之間往來通訊的類型與時間，供使用者事後稽核。
- **FR-012**: 系統 MUST 讓使用者登記機器名稱與硬體位址後，對該名稱發送區域網路
  喚醒訊號。
- **FR-013**: 系統 MUST 讓使用者登記地點後查詢並顯示目前與未來數天的天氣資訊，且
  同一地點的查詢結果 SHOULD 在短時間內重複使用快取結果而非每次都重新查詢外部服務。
- **FR-014**: 系統 MUST 提供 Markdown 筆記的建立、編輯、儲存能力，且筆記內容 MUST
  可分別透過終端機介面與瀏覽器介面存取與編輯。
- **FR-015**: 系統 MUST 讓使用者下載線上音樂資源為本機音樂檔案，並在瀏覽器介面中
  提供清單、播放、刪除等管理能力。
- **FR-016**: 系統 MUST 讓使用者產生對應到本機（或指定機器）網頁介面連結的 QR
  Code，供其他裝置掃描後快速開啟連線。
- **FR-017**: 系統 MUST 讓使用者登記本機一或多個 git 專案路徑，並查詢這些專案目前
  是否存在未提交的變更。
- **FR-018**: 系統 MUST 提供自我更新能力：使用者下達更新指令後，系統於背景取得
  最新版本並重新建置，僅在建置成功後才切換到新版本運作；建置失敗則維持現行版本。
- **FR-019**: 使用者對遠端機器（同群組或跨網域）下達更新指令時，更新流程 MUST 在
  該遠端機器上執行，而非在發出指令的本機執行。
- **FR-020**: 系統 MUST 在無法以終端機前景方式運作時（例如非互動式啟動），仍持續
  提供瀏覽器介面與背景裝置回報等核心能力，不因缺少終端機前景而停止服務。

### Key Entities

- **Device（裝置）**：群組中的一台機器，屬性包含識別碼、內網位址、作業系統、版本、
  目前角色（獨立/伺服器/客戶端）、運作時間、最近回報時間。
- **Domain（網域/群組）**：由一台群組伺服器與其客戶端組成的集合；多個網域可透過共同
  的橋接代號互相連結，形成可跨網域互相可見與操控的更大群組。
- **Plugin / Panel（功能模組／面板）**：一項可獨立啟用的功能單元，擁有自己的指令
  集合，並可選擇性提供一個可視面板；面板具有位置、大小、顯示狀態等屬性。
- **Remote Request/Reply（遠端請求／回應）**：使用者跨機器或跨網域下達指令、查詢
  面板、傳輸檔案時產生的一組請求與對應回應，經加密傳輸並以時間戳防止逾時或重放。
- **Activity Log Entry（活動紀錄項目）**：一筆本機與其他機器或外部服務之間往來的
  紀錄，含類型與時間。
- **File Transfer Job（檔案複製工作）**：使用者發起的一次資料夾內容同步工作，含
  來源、目的、目前進度。
- **WOL Target（喚醒目標）**：使用者登記的一台機器名稱與其硬體位址。
- **Weather Location（天氣地點）**：使用者登記查詢天氣的城市或地點。
- **Git Watch Entry（專案監控項目）**：使用者登記要監控的本機 git 專案路徑與其目前
  是否有未提交變更。
- **Note（筆記）**：一份使用者建立的 Markdown 文字內容，可被本機終端機或瀏覽器
  介面讀取與編輯。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 使用者可以任選三種介面（終端機文字指令列、終端機圖形化面板、瀏覽器）
  中的任一種完成日常操作，切換介面後看到的狀態 100% 與切換前一致，不需要重新設定
  或重新查詢。
- **SC-002**: 兩台彼此網路無法直接連線的機器，在雙方都已佈署相同共用金鑰與橋接
  代號的前提下，使用者可以完成一次跨網域指令執行並取得回應，且過程中不需要任一方
  對外開放連接埠。
- **SC-003**: 超過允許時間窗或屬於重複傳送的跨網域訊息，100% 被系統拒絕，不會被
  誤判為有效指令而執行。
- **SC-004**: 群組內客戶端機器在啟動後，於兩個回報週期內，群組中其他機器都能在
  裝置清單查到它的上線資訊。
- **SC-005**: 使用者對本機下達自我更新指令時，既有服務（瀏覽器介面、裝置回報）
  在整個更新流程中持續可用，不因更新而中斷；只有在新版本成功建置後才會發生一次
  短暫的服務切換。
- **SC-006**: 使用者透過登記的機器名稱（而非硬體位址）即可成功喚醒對應機器。
- **SC-007**: 檔案複製工作在重複執行時，內容已相同的檔案 100% 被跳過，不會產生
  重複傳輸或覆蓋既有檔案的情況。

## Assumptions

- 使用者本人同時是這套系統所有機器的維運者，具備在每台機器上手動佈署共用金鑰
  檔案與啟動設定的權限與能力。
- 所有需要跨網域互通的機器，都能對外連線到系統選用的公開訊息橋接服務；若該服務
  不可用，跨網域相關能力會暫時無法使用，但不影響單機與同群組內的功能。
- 使用情境為使用者本人與少量受信任機器之間的個人化維運，不涉及多租戶或多使用者
  權限隔離設計。
- 檔案複製功能目前僅開放特定用途的資料夾（音樂、筆記），不代表可任意複製機器上
  任意路徑的檔案。
